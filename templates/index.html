Federated Learning for Brain Tumor Detection

Welcome to our comprehensive project on federated learning for brain tumor detection using MRI scans. This project aims to leverage distributed data across multiple hospitals while preserving data privacy. By utilizing advanced techniques like diffusion models for data augmentation, we strive to improve the accuracy and robustness of tumor detection models.

Project Overview

The primary objective of this project is to develop a federated learning system that can detect brain tumors from MRI scans. The system is designed to work with data distributed across various fictional hospitals, ensuring that sensitive medical images remain local and private. This approach not only enhances data privacy but also allows for a more diverse dataset, improving the model's generalization capabilities.

Dataset Description

Our dataset is sourced from Kaggle and consists of MRI scans categorized into four types:

  • Glioma Tumors
  • Meningioma Tumors
  • Pituitary Tumors
  • Non-Tumor Cases

The original dataset contains 7023 images, with 5712 images for training and 1311 images for testing. This dataset provides a comprehensive foundation for training our models.

Final Preprocessed Dataset

After applying data augmentation using the diffusion model, the final preprocessed dataset consists of a total of 7616 images in the training set. This augmentation helps to balance the dataset and improve the model's ability to generalize across different tumor types. Below is the distribution of images in each category:

  • Glioma Tumors: 2000 images (1500 original, 500 augmented)
  • Meningioma Tumors: 1700 images (1300 original, 400 augmented)
  • Pituitary Tumors: 1500 images (1200 original, 300 augmented)
  • Non-Tumor Cases: 2416 images (1712 original, 704 augmented)

The augmentation process has been carefully designed to ensure that each category is well-represented in the training dataset.

Diffusion Model for Data Augmentation

To address class imbalance and enhance model performance, we employ a diffusion model to generate synthetic images. This process involves generating synthetic images, which make up 25% of the final training dataset. The diffusion model helps in creating a more balanced and diverse dataset, which is crucial for training robust machine learning models.

Federated Learning Process

Our federated learning system simulates a network of six fictional hospitals, each acting as a client:

  • Green Valley Hospital
  • Silver Lake Medical Center
  • Blue Ridge Health Institute
  • Sunnydale Community Hospital
  • Maplewood General Hospital
  • Riverbend Specialty Clinic

Training rounds involve an initial round to establish a baseline, followed by subsequent rounds with varying client participation. This approach ensures that the model benefits from diverse data sources while maintaining data privacy.
