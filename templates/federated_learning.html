Federated Learning for Brain Tumor Detection

Discover how federated learning enables privacy-preserving model training across distributed datasets.

Introduction to Federated Learning

Federated learning is a decentralized approach to machine learning where models are trained across multiple devices or servers holding local data samples, without exchanging them. This method is particularly beneficial in scenarios where data privacy is paramount, such as in medical imaging.

Client Simulation

In our project, we simulate a network of six fictional hospitals, each acting as a client in the federated learning system. These clients collaborate to train a global model while keeping their data local:

  • Green Valley Hospital
  • Silver Lake Medical Center
  • Blue Ridge Health Institute
  • Sunnydale Community Hospital
  • Maplewood General Hospital
  • Riverbend Specialty Clinic

Each client trains the model on its local dataset and shares only the model updates with a central server, ensuring that sensitive medical data remains private.

Training Rounds

The federated learning process is organized into multiple training rounds:

  • Initial Round: The global model is trained on 10% of the dataset to establish a baseline performance.
  • Subsequent Rounds: Clients participate in training rounds, with the number of participating clients varying each round. This simulates real-world scenarios where clients may not always be available.

After each round, the central server aggregates the model updates using the Federated Averaging (FedAvg) algorithm to update the global model.

Evaluation Metrics

To assess the performance of the federated learning model, we use several key metrics:

  • Accuracy: The ratio of correct predictions to total predictions.
  • Precision: The ratio of true positives to the sum of true positives and false positives.
  • Recall: The ratio of true positives to the sum of true positives and false negatives.
  • F1 Score: The harmonic mean of precision and recall.
  • Root Mean Squared Error (RMSE): A measure of the differences between predicted and actual values.
  • Confusion Matrix: A table showing true vs. predicted classifications.

These metrics are calculated after each round to evaluate both local and global model performance.

Benefits of Federated Learning

Federated learning offers several advantages in the context of medical imaging:

  • Data Privacy: Sensitive medical data remains local, reducing the risk of data breaches.
  • Scalability: The system can easily scale to include more clients without compromising data security.
  • Diversity: The model benefits from diverse datasets across different clients, improving its generalization capabilities.
